**项目背景介绍（2-5 分钟）**
```
首先，我们来了解一下为什么要开发这个 SDK。

微信支付是电商平台不可或缺的一部分，但是：
- 官方文档分散，不易理解
- 签名、加密等安全处理复杂
- 异常处理需要统一规范
- 每次开发都要重复造轮子

我们的目标就是：
1. 封装复杂的微信支付 API
2. 提供简单易用的接口
3. 统一异常处理
4. 完善的文档和示例

最终实现：一行代码完成支付对接！
```

**项目架构设计（5-15 分钟）**
```
现在，让我们来看一下项目的整体架构。

【展示项目结构图】

整个项目采用模块化设计：

1. **wechat-starter-core** - 核心模块
   - 配置管理
   - 安全工具（签名、加密）
   - 数据工具（JSON、时间）
   - 网络工具（HTTP 客户端）

2. **wechat-starter-pay** - 支付模块
   - JSAPI 支付
   - Native 支付
   - APP 支付
   - 退款
   - 分账
   - 商家转账
   - 红包

3. **wechat-starter-auth** - 认证模块
   - 小程序登录
   - 公众号授权
   - 手机号解密

4. **wechat-starter-message** - 消息模块
   - 模板消息
   - 订阅消息

5. **wechat-starter-iot** - IoT 模块
   - 设备管理
   - 传感器数据处理

这种模块化设计的好处：
- 职责清晰，易于维护
- 可以按需引入
- 便于扩展

接下来，我们会逐个模块进行开发。
```

### 第 3 集：核心模块开发 - 配置管理

**设计配置类（0-10 分钟）**
```
配置管理是 SDK 的基础。

我们需要设计一个灵活的配置类，支持：
- 商户号配置
- API 密钥配置
- 证书配置
- 回调地址配置
- 沙箱环境开关

【展示 WechatProperties 设计】

使用 Lombok 的 @Data 注解自动生成 getter/setter
使用静态内部类组织不同模块的配置
```

**实现自动配置（10-20 分钟）**
```
【展示 WechatAutoConfiguration】

Spring Boot 的自动配置原理：
1. @Configuration 标注配置类
2. @Bean 标注要注入的 bean
3. @Conditional 条件加载

这样用户只需要在 application.yml 中配置，
Spring Boot 会自动创建所需的 bean！
```

**配置测试（20-25 分钟）**
```
【演示配置加载】

编写测试验证：
- 配置是否正确加载
- 默认值是否生效
- 环境变量是否覆盖

好的，配置管理完成！
```
